# Create Architecture

## Use
Produce or update architecture for the repository or feature in scope.

## Inputs
- Source specification, RFC, or problem statement
- Scope boundaries
- Constraints and existing architecture context

## Instructions
- Apply `.ai/skills/architecture-management/SKILL.md`.
- Read current architecture docs and ADRs before proposing change.
- Prefer the simplest design that satisfies the requirement.
- Create or update `architecture.md`.
- Add ADRs in `docs/decisions/` for material decisions.

## Check
- Components have clear responsibilities.
- Tradeoffs are explicit.
- Dependencies are justified.
