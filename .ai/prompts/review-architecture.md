# Review Architecture

## Use
Review existing architecture for correctness, drift, and missing decisions.

## Inputs
- Current `architecture.md`
- Relevant diffs or recent changes
- Known concerns, if any

## Instructions
- Apply `.ai/skills/architecture-management/SKILL.md`.
- Focus on structure, boundaries, and dependencies.
- Report findings as confirmed, drift, gaps, and recommendations.
- Update ADRs when a decision is deprecated or superseded.

## Check
- Findings cite the architecture they refer to.
- Recommendations include the tradeoff.
- The review is actionable.
